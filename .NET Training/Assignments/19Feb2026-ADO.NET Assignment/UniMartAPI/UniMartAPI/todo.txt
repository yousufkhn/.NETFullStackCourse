## Critical Bugs (Fix These First)

### 1. `DeleteProduct` has NO ownership check
You have it in `UpdateProduct` but completely missed it in `DeleteProduct`:
```csharp
// ProductsController.cs — DeleteProduct
var product = await _service.GetByIdAsync(id);
if (product == null) return NotFound();

// ← NO ownership check here
// Any logged-in user can delete anyone else's product right now
var success = await _service.DeleteAsync(id);
```
The fix is one line — add `if (product.UserId != userId) return Forbid();` before `DeleteAsync`.

### 2. `ExceptionMiddleware` is completely empty

```csharp
public class ExceptionMiddleware

    // nothing here

```
This is completely unregistered and doing nothing. This was on your to-do list.

### 3. Hard Delete instead of Soft Delete
`DeleteAsync` does `_context.Products.Remove(product)` — permanently wipes the row. But your model already has `IsActive` specifically for this. Just set `product.IsActive = false` and save.

## Design Issues (Medium Priority)

### 4. `CreateProductDto` has zero validation
`RegisterDto` is well validated, but `CreateProductDto` has nothing:
```csharp
public class CreateProductDto
{
    public string Name { get; set; }       // no [Required], no length limit
    public string Description { get; set; } // no length limit
    public decimal Price { get; set; }     // no [Range] — negative price is accepted
    public int CategoryId { get; set; }    // no validation
}
```

### 5. `AuthController` directly uses `DbContext` — bypasses service layer
`Register` and `Login` call `_context` directly in the controller. All DB logic should live in `AuthService`. Right now `AuthService` only has `GetProfileAsync` — Register and Login should move into it too.

### 6. `Login` is not `async`
```csharp
public IActionResult Login(LoginDto dto)  // ← synchronous
{
    var user = _context.Users.SingleOrDefault(...); // ← blocking DB call
```
Should be `public async Task<IActionResult> Login` with `await _context.Users.SingleOrDefaultAsync(...)`.

### 7. `Register` uses synchronous `.Any()`
```csharp
if (_context.Users.Any(u => u.Email == dto.Email)) // ← blocks the thread
```
Should be `await _context.Users.AnyAsync(...)`.

### 8. `UpdateAsync` accepts raw `Product` model instead of a DTO
```csharp
public async Task<IActionResult> UpdateProduct(int id, Product updatedProduct)
```
You're exposing your domain model directly as input. Someone could send `UserId`, `CreatedAt`, `IsActive` and potentially overwrite them. Needs an `UpdateProductDto` with only the fields a user should be allowed to change (`Name`, `Description`, `Price`, `CategoryId`).

### 9. `UpdateAsync` only updates `Name` and `Price`
```csharp
existing.Name = product.Name;
existing.Price = product.Price;
// Description and CategoryId are never updated
```
`Description` and `CategoryId` changes are silently ignored.

### 10. `pageSize` has no upper bound
```csharp
int pageSize = 10
```
Someone can call `?pageSize=100000` and dump the entire database in one request. Add a cap: `if (pageSize > 50) pageSize = 50;` or return `BadRequest`.
